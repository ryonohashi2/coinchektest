# Finance Dashboard デザインシステム

このドキュメントでは、Finance DashboardアプリケーションのデザインシステムとUIガイドラインを定義します。

## デザイン原則

### 1. シンプルさ
- 情報の階層を明確にし、ユーザーが迷わないインターフェース
- 不要な装飾を排除し、機能性を重視

### 2. 一貫性
- 全体を通して統一されたビジュアル言語
- 予測可能なインタラクションパターン

### 3. アクセシビリティ
- WCAG 2.1 AA準拠
- キーボードナビゲーション対応
- 適切なコントラスト比の確保

### 4. レスポンシブ
- モバイルファーストアプローチ
- 様々なデバイスサイズに対応

## シャドウ

### レベル
- **Level 1**: `shadow-sm` - カード、ボタン
- **Level 2**: `shadow` - ドロップダウン
- **Level 3**: `shadow-lg` - モーダル
- **Level 4**: `shadow-xl` - ポップオーバー

## ボーダー半径

- **Small**: `rounded-sm` (2px) - 小さな要素
- **Default**: `rounded` (4px) - ボタン、入力フィールド
- **Medium**: `rounded-md` (6px) - カード
- **Large**: `rounded-lg` (8px) - モーダル
- **Full**: `rounded-full` - アバター、バッジ

## アニメーション

### トランジション
- **Fast**: `150ms` - ホバー、フォーカス
- **Normal**: `300ms` - 状態変化

### イージング
- **Ease Out**: `cubic-bezier(0, 0, 0.2, 1)` - 標準
- **Ease In**: `cubic-bezier(0.4, 0, 1, 1)` - 消失
- **Ease In Out**: `cubic-bezier(0.4, 0, 0.2, 1)` - 双方向

## コンポーネント設計原則

### Atomic Design
1. **Atoms**: ボタン、入力フィールド、ラベル
2. **Molecules**: 検索ボックス、カード
3. **Organisms**: ヘッダー、ナビゲーション
4. **Templates**: ページレイアウト
5. **Pages**: 具体的なページ

### 状態管理
- **Default**: 通常状態
- **Hover**: マウスオーバー
- **Focus**: フォーカス状態
- **Active**: アクティブ状態
- **Disabled**: 無効状態
- **Loading**: 読み込み状態
