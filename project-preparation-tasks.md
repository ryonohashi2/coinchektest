# プロジェクト準備タスク一覧

## EPIC 1: プロジェクト基盤・運用整備

### User Story 1-1: 開発環境・運用ルールの統一
- **目的**: チーム全員が同一の開発・運用ルールで作業できるようにする
- **成果物**: セットアップ手順書、運用ルールドキュメント
- **担当者**: PM, シニアエンジニア
- **想定工数**: 0.5日
- **優先度**: S

#### タスク
1. Node.js, Yarn, Git のバージョン統一・セットアップ手順書作成
2. .env.local, .gitignore, セキュリティ運用ルールの明文化
3. コード規約・Lint/Formatルール策定（eslint, prettier等）
4. README, アーキテクチャ・ディレクトリ構成ドキュメント整備

## 依存関係マッピング

- **EPIC 1**（基盤・運用整備）は全てのEPICの前提
- **CI/CD・Lint/Format・セキュリティ運用ルール**は最優先
- **ディレクトリ構成・雛形作成**は、以降の実装タスクの基盤
- **API仕様調査・型定義**は、API実装・UI実装の前提
- **デザイン/UX基盤**はUI実装の前提

---

### 優先度について
- S: 開発着手前に必須
- A: 早期に着手推奨
